# В чем разница между изменяемыми (mutable) и неизменяемыми (immutable) объектами в JavaScript?

Объект в JavaScript **изменяемый**. Его свойства можно менять после создания.

```js
const user = { name: 'Ann' };
user.name = 'Bob';
```

**Неизменяемость** означает, что значение нельзя поменять на месте. Вместо этого создают новую версию.

Важно:

* объекты и массивы — изменяемые;
* примитивы `string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint` — неизменяемые;
* `const` запрещает переназначение переменной, но не делает объект неизменяемым.

```js
const user = { name: 'Ann' };
user.name = 'Bob'; // работает
// user = {}        // ошибка
```

Неизменяемый подход часто используют в React и state management. Он упрощает сравнение старого и нового состояния.

Короткий ответ для собеседования:

> В JavaScript объекты по умолчанию изменяемые. Это значит, что их свойства можно менять после создания. Неизменяемый подход строится на создании нового объекта вместо изменения старого. `const` защищает ссылку, а не содержимое объекта.

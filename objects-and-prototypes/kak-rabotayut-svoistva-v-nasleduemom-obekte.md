# Как работают свойства в наследуемом объекте?

Поведение зависит от операции.

#### Чтение

Если свойство есть в объекте, берут его. Если нет, ищут в прототипе.

#### Изменение

Если это обычное свойство из прототипа, то при записи создаётся новое собственное свойство в текущем объекте.

#### Удаление

`delete` удаляет только собственное свойство объекта. Свойство в прототипе он не трогает.

```js
const animal = { eats: true };
const dog = Object.create(animal);

dog.eats; // true

dog.eats = false; // создаётся собственное свойство

delete dog.eats; // удаляется только dog.eats
console.log(dog.eats); // снова true из прототипа
```

Короткий ответ для собеседования:

> При чтении JavaScript ищет свойство по цепочке прототипов. При записи обычно создаётся собственное свойство объекта. `delete` удаляет только собственные свойства и не удаляет данные из прототипа.

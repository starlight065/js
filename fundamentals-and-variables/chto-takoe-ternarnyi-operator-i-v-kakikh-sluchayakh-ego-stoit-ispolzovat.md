# Что такое тернарный оператор и в каких случаях его стоит использовать?

Тернарный оператор — это короткая форма условного выражения, которая позволяет выбрать одно из двух значений в зависимости от условия.

В JavaScript он записывается так:

```jsx
condition ? valueIfTrue : valueIfFalse
```

Пример:

```jsx
const age = 18;

const access = age >= 18 ? "Allowed" : "Denied";

console.log(access); // "Allowed"
```

То есть это аналог простого `if...else`:

```jsx
let access;

if (age >= 18) {
  access = "Allowed";
} else {
  access = "Denied";
}
```

### Когда стоит использовать

Тернарный оператор хорошо использовать, когда нужно **быстро выбрать одно значение из двух вариантов**.

Например:

```jsx
const theme = isDarkMode ? "dark" : "light";
```

Или в JSX:

```jsx
<button>
  {isLoading ? "Loading..." : "Submit"}
</button>
```

### Когда лучше не использовать

Не стоит использовать тернарный оператор, если условие становится сложным или внутри появляются дополнительные действия.

Плохо читается:

```jsx
const result = score > 90
  ? "Excellent"
  : score > 70
    ? "Good"
    : score > 50
      ? "Average"
      : "Fail";
```

В таком случае лучше использовать `if...else` или вынести логику в отдельную функцию.

### Главное правило

Тернарный оператор стоит использовать для **простых условий и выбора значения**, а не для сложной бизнес-логики.

Короткий ответ для собеседования:

> Тернарный оператор — это сокращённая форма `if...else`, которая возвращает одно из двух значений в зависимости от условия. Его удобно использовать для простых условий, например при присваивании значения или условном рендеринге. Если логика становится сложной или ухудшается читаемость, лучше использовать обычный `if...else`.

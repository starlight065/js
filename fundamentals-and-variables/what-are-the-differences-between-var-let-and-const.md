# What are the differences between var, let, and const?

#### Область видимости (Scope)

* **`var` имеет функциональную область видимости (function scope).**\
  Переменная, объявленная внутри функции, видна только там. Но если она объявлена внутри блока (например, `if` или `for`), она будет видна и **за пределами** этого блока.
* **`let` и `const` имеют блочную область видимости (block scope).**\
  Они видны только внутри того блока кода (ограниченного фигурные скобками `{}`), в котором они были объявлены.

```jsx
if (true) {
  var a = 'Я var';
  let b = 'Я let';
  const c = 'Я const';
}

console.log(a); // Выведет: "Я var" (вышла за пределы блока if)
console.log(b); // ReferenceError: b is not defined
console.log(c); // ReferenceError: c is not defined
```

При переопределении и повторном объявлении переменных их поведение напрямую зависит от ключевого слова. Переменную, объявленную через **`var`**, можно повторно объявить в том же окружении и изменить её значение (переприсвоить). Переменную **`let`** нельзя объявить повторно в той же области видимости, но её значение можно менять. Для **`const`** правила самые строгие: её нельзя объявить повторно или перезаписать значение; при этом значение нужно задать сразу при создании переменной.

#### Поднятие (Hoisting) и Временная мертвая зона (TDZ)

Все переменные в JS "поднимаются" (hoisting) в начало своей области видимости перед выполнением кода. Но ведут они себя по-разному.

* **`var`:** Поднимается и автоматически инициализируется значением `undefined`. К ней можно обратиться до объявления.
* **`let` и `const`:** Поднимаются, но не инициализируются. Они находятся в «временной мертвой зоне» (Temporal Dead Zone) до тех пор, пока интерпретатор не дойдет до строки с их объявлением. Обращение к ним до этого момента вызовет ошибку.

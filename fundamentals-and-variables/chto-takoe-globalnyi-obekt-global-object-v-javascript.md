# Что такое глобальный объект (global object) в JavaScript?

**Глобальный объект** в JavaScript — это специальный объект, который существует всегда, в любой среде выполнения (в браузере, на сервере Node.js и т.д.), и доступен из любой точки вашей программы без какого-либо импорта.

Он служит «контейнером» для глобальных переменных, функций, а также для встроенных возможностей самого языка и платформы.

* **В браузерах:** `window`.
* **В Node.js (на сервере):** `global`.
* **В Web Workers:** `self`.

**Единый стандарт: `globalThis`**

Чтобы решить проблему разницы в именах, в современном стандарте JavaScript (ES2020) было введено единое имя — **`globalThis`**. Оно работает везде: и в браузере, и в Node.js, и в воркерах.

| **Категория**                              | **Подкатегория / Среда** | **Примеры сущностей**                                                                           |
| ------------------------------------------ | ------------------------ | ----------------------------------------------------------------------------------------------- |
| **1. Встроенные функции и значения языка** | Значения                 | `undefined`, `NaN`, `Infinity`                                                                  |
|                                            | Функции                  | `parseInt()`, `parseFloat()`, `isNaN()`, `encodeURIComponent()` ...                             |
| **2. Встроенные классы и объекты**         | Классы и структуры       | `Object`, `Array`, `Function`, `String`, `Number`, `Boolean`, `Promise`, `Map`, `Set`, `RegExp` |
|                                            | Утилитарные объекты      | `Math`, `JSON`                                                                                  |
| **3. API конкретной среды выполнения**     | В браузере (Web API)     | `document`, `fetch()`, `localStorage`, `setTimeout()`, `alert()`                                |
|                                            | В Node.js                | `process`, `require()`, `module`                                                                |

Если вы объявляете переменную с помощью `var` в глобальном контексте, она автоматически **становится свойством глобального объекта**

Переменные, объявленные с помощью `let` или `const` в глобальном контексте, **НЕ становятся свойствами глобального объекта**. Они находятся в специальной внутренней области видимости (называемой декларативной областью видимости)

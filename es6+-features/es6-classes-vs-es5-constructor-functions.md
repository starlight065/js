# ES6 classes vs ES5 constructor functions

С точки зрения механики они очень близки. Оба варианта работают через прототипы.

Разница в основном в синтаксисе и удобстве.

* `class` читается проще;
* `extends` и `super` делают наследование удобнее;
* методы класса автоматически попадают в `prototype`;
* классы нельзя вызвать без `new`.

```js
function UserES5(name) {
  this.name = name;
}
UserES5.prototype.sayHi = function () {};

class UserES6 {
  constructor(name) {
    this.name = name;
  }

  sayHi() {}
}
```

Короткий ответ для собеседования:

> ES6-классы — это синтаксический сахар над функциями-конструкторами и прототипами ES5. Поведение похоже, но `class` делает код чище, безопаснее и удобнее для наследования.

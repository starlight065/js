# Что такое optional chaining и когда это полезно?

Опциональная цепочка `?.` позволяет безопасно обращаться к вложенным свойствам объекта.

Если слева `null` или `undefined`, код не упадёт с ошибкой. Он просто вернёт `undefined`.

Пример:

```js
const user = {};

console.log(user.profile?.address?.city);
```

Без `?.` такой код может выбросить ошибку, если `profile` не существует.

Это полезно, когда:

* данные приходят с сервера;
* структура объекта может быть неполной;
* нужно сократить длинные проверки через `&&`.

Короткий ответ для собеседования:

> Опциональная цепочка `?.` нужна для безопасного доступа к вложенным свойствам. Если промежуточного значения нет, выражение вернёт `undefined`, а не ошибку. Это удобно при работе с неполными объектами и ответами API.

# В чем разница между == и === в JavaScript?

#### **`==` — Нестрогое сравнение (с приведением типов)**

Этот оператор сравнивает только **значения**, предварительно пытаясь привести их к общему типу (это называется _приведением типов_ или _coercion_).

Если типы сравниваемых элементов разные, JavaScript автоматически преобразует один из них, чтобы они совпали, и только потом сравнивает.

* `'5' == 5` ➡️ `true` (строка `'5'` преобразуется в число `5`).
* `true == 1` ➡️ `true` (булево значение `true` преобразуется в число `1`).
* `0 == false` ➡️ `true` (оба значения приводятся к ложному типу/числу `0`).
* `null == undefined` ➡️ `true` (это особое правило в JS).

#### **`===` — Строгое сравнение (без приведения типов)**

Этот оператор сравнивает и **значение**, и **тип данных**. Если типы разные, он сразу возвращает `false`, даже не пытаясь оценивать значения.

**Примеры:**

* `'5' === 5` ➡️ `false` (тип слева — строка, справа — число).
* `true === 1` ➡️ `false` (слева — boolean, справа — number).
* `0 === false` ➡️ `false` (разные типы).
* `null === undefined` ➡️ `false` (разные типы).
* `5 === 5` ➡️ `true` (и типы, и значения одинаковые).

# Объясни особенности поверхностного копирования (shallow copy) для массивов объектов.

Поверхностная копия массива создаёт **новый массив**, но его элементы остаются теми же ссылками.

Это особенно важно для массивов объектов. Сам массив новый, а объекты внутри — старые.

```js
const users = [{ name: 'Ann' }, { name: 'Bob' }];
const copy = [...users];

copy[0].name = 'Kate';

console.log(users[0].name); // 'Kate'
```

Что важно понимать:

* `[...]`, `slice()` и `Array.from()` делают только поверхностную копию;
* изменение самого массива не затрагивает оригинал;
* изменение вложенного объекта затрагивает оба массива.

Если нужна независимая копия объектов, их нужно копировать отдельно.

```js
const deepEnough = users.map(user => ({ ...user }));
```

Короткий ответ для собеседования:

> Поверхностное копирование массива создаёт новый массив, но не копирует вложенные объекты. Поэтому изменения объекта внутри копии будут видны и в оригинале. Для массива объектов это частая ловушка.

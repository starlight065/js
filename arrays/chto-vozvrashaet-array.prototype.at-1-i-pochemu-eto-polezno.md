# Что возвращает Array.prototype.at(-1) и почему это полезно?

`at(-1)` возвращает **последний элемент массива**.

```js
const arr = ['a', 'b', 'c'];

console.log(arr.at(-1)); // 'c'
```

Почему это удобно:

* код читается лучше, чем `arr[arr.length - 1]`;
* можно использовать отрицательные индексы;
* особенно удобно при доступе к элементам с конца.

Если элемента нет, метод вернёт `undefined`.

```js
console.log([].at(-1)); // undefined
```

На собеседовании полезно сравнить:

```js
arr[arr.length - 1]
arr.at(-1)
```

Оба варианта работают, но `at()` делает намерение понятнее.

Короткий ответ для собеседования:

> `arr.at(-1)` возвращает последний элемент массива. Это удобно, потому что не нужно писать `arr[arr.length - 1]`. Метод делает код короче и позволяет использовать отрицательные индексы для доступа с конца.

# Назови 3 мутирующих (mutating) и 3 немутирующих (non-mutating) метода массивов.

**Мутирующие** методы меняют исходный массив.

Примеры:

* `push()`
* `splice()`
* `sort()`

**Немутирующие** методы не меняют исходный массив. Они возвращают новый массив или новое значение.

Примеры:

* `slice()`
* `concat()`
* `map()`

```js
const arr = [3, 1, 2];
const copy = arr.slice();

arr.sort();
console.log(arr);  // [1, 2, 3]
console.log(copy); // [3, 1, 2]
```

Что важно проговорить:

* мутирующие методы опасны, если массив используется в нескольких местах;
* немутирующие методы удобнее для предсказуемого кода;
* в современном JavaScript всё чаще предпочитают немутирующие варианты, например `toSorted()`.

На собеседовании можно дополнительно назвать `pop()`, `shift()`, `unshift()`, `reverse()`, `filter()`, `flat()` и `toSorted()`.

Короткий ответ для собеседования:

> К мутирующим методам относятся `push()`, `splice()` и `sort()`. К немутирующим — `slice()`, `concat()` и `map()`. Ключевая идея — одни меняют исходный массив, а другие работают без побочного эффекта.

# Каково поведение sort() по умолчанию без компаратора и почему оно может быть некорректным для чисел?

По умолчанию `sort()` сравнивает элементы **как строки**.

Это лексикографическая сортировка, а не числовая.

```js
const numbers = [1, 10, 2];
numbers.sort();

console.log(numbers); // [1, 10, 2]
```

Почему так происходит:

* значения приводятся к строкам;
* сравнение идёт посимвольно;
* `'10'` идёт раньше `'2'`.

Для чисел нужен компаратор:

```js
const numbers = [1, 10, 2];
numbers.sort((a, b) => a - b);

console.log(numbers); // [1, 2, 10]
```

Для сортировки по убыванию компаратор обычно пишут так:

```js
numbers.sort((a, b) => b - a);
```

Важно помнить, что `sort()` мутирует исходный массив. Если менять исходный массив нельзя, удобнее использовать `toSorted()`.

Короткий ответ для собеседования:

> Без компаратора `sort()` сортирует элементы как строки, поэтому числовой порядок может быть неправильным. Для чисел обычно передают `(a, b) => a - b`. Ещё важно помнить, что `sort()` меняет исходный массив.

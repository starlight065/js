# В чем разница между методами map(), filter() и reduce()?

Эти методы решают разные задачи.

* `map()` преобразует каждый элемент.
* `filter()` оставляет только подходящие элементы.
* `reduce()` сворачивает массив в одно итоговое значение.

```js
const numbers = [1, 2, 3, 4];

const doubled = numbers.map(n => n * 2);      // [2, 4, 6, 8]
const even = numbers.filter(n => n % 2 === 0); // [2, 4]
const sum = numbers.reduce((acc, n) => acc + n, 0); // 10
```

Как отличать их на собеседовании:

* `map()` — на выходе всегда массив той же длины.
* `filter()` — на выходе массив той же или меньшей длины.
* `reduce()` — на выходе может быть что угодно: число, объект, массив, `Map`.

Короткий ответ для собеседования:

> `map()` меняет форму каждого элемента, `filter()` отбирает элементы по условию, а `reduce()` агрегирует массив в одно значение. Если нужен новый массив той же длины — `map()`. Если нужен отбор — `filter()`. Если нужен итоговый результат — `reduce()`.

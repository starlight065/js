# В чем разница между Array.prototype.forEach() и Array.prototype.map()?

Оба метода проходят по массиву.

Разница в цели:

* `forEach()` используют для **побочных эффектов**.
* `map()` используют для **преобразования данных**.

`forEach()` ничего полезного не возвращает. Он возвращает `undefined`.

`map()` возвращает **новый массив** той же длины. Каждый элемент в нём — результат функции коллбека.

```js
const numbers = [1, 2, 3];

numbers.forEach(n => console.log(n));

const doubled = numbers.map(n => n * 2);
console.log(doubled); // [2, 4, 6]
```

Что важно помнить:

* `forEach()` не подходит, если нужен результат вычисления.
* `map()` удобно чейнить с `filter()` и другими методами.
* `map()` не меняет исходный массив, если коллбек сам ничего не мутирует.

Короткий ответ для собеседования:

> `forEach()` нужен, когда надо выполнить действие для каждого элемента, например логирование или отправку данных. `map()` нужен, когда надо получить новый массив из старого. Главное отличие — `map()` возвращает новый массив, а `forEach()` возвращает `undefined`.

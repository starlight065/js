# В чем разница между модулем и пространством имен (namespace) в JavaScript?

## Главное отличие

* Модуль (ES Modules) — это современный стандарт JS, где каждый файл — изолированная область видимости. Переменные не попадают в глобальную среду, а связи между файлами управляются через `import` / `export`.
* Пространство имен (Namespace) — это устаревший паттерн (или фича TypeScript), где код группируется внутри одного глобального объекта (например, `window.MyApp`).

#### Шпаргалка по критериям

* Область видимости:
  * В модулях — строгая изоляция на уровне файла.
  * В namespaces — объект находится в глобальной видимости.
* Управление зависимостями:
  * В модулях — автоматическое (движок сам понимает, что откуда импортируется).
  * В namespaces — ручное (нужно строго следить за порядком подключения `<script>` в HTML).
* Оптимизация (Tree Shaking):
  * Модули отлично оптимизируются сборщиками (удаляется неиспользуемый код).
  * Из объектов (namespaces) неиспользуемый код удалить практически невозможно.

#### Что отвечать, если спросят про TypeScript namespaces?

> «В TypeScript есть встроенное ключевое слово `namespace`, но сами создатели TS сейчас рекомендуют использовать стандартные ES-модули. Namespaces остались в основном для поддержки старого кода или декларации типов».

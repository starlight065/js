# Передача параметров происходит по значению (pass-by-value) или по ссылке (pass-by-reference)?

* **Коротко:** в JavaScript аргументы всегда передаются **по значению**.
* **Примитивы** (`number`, `string`, `boolean`, `null`, `undefined`, `symbol`, `bigint`) передаются как копия значения.

```jsx
let a = 10;

function change(x) {
  x = 20;
}

change(a);
console.log(a); // 10
```

Переменная `a` не изменилась, потому что `x` — это отдельная копия значения.

* **Объекты** тоже передаются по значению, но само значение — это **ссылка на объект**.

```jsx
const user = { name: "Alex" };

function change(obj) {
  obj.name = "Max";
}

change(user);
console.log(user.name); // "Max"
```

Здесь функция получила копию ссылки, но эта ссылка указывает на тот же объект. Поэтому через неё можно изменить содержимое объекта.

* Но если внутри функции присвоить параметру новый объект, внешняя переменная не изменится:

```jsx
const user = { name: "Alex" };

function change(obj) {
  obj = { name: "Max" };
}

change(user);
console.log(user.name); // "Alex"
```

Почему? Потому что `obj` — это локальная переменная внутри функции. Мы просто положили в неё новую ссылку, но внешняя переменная `user` продолжает ссылаться на старый объект.

### Итог

JavaScript — это **pass-by-value**.

Для объектов передается **копия ссылки**, поэтому:

* можно менять содержимое объекта;
* нельзя заменить внешний объект через параметр функции.

Такое поведение часто называют **call-by-sharing**.

# В чем разница между Map, Set, WeakMap и WeakSet?

* `Map` — это продвинутый объект (коллекция "ключ-значение"), где ключом может быть [_что угодно_](#user-content-fn-1)[^1].
* `Set` — это коллекция _уникальных значений_ (массив без повторений).

#### WeakMap

* Ключи: Только объекты. Примитивы (строки, числа) быть ключами не могут. Значения могут быть любыми.
* Сборка мусора: Если на объект-ключ больше нет других ссылок в программе, кроме как внутри `WeakMap`, этот объект автоматически удаляется из памяти сборщиком мусора вместе со своим значением.
* Перебираемость: Нельзя перебрать (`for...of` не работает, методов `keys()`, `values()` нет). Сборщик мусора работает асинхронно, поэтому JavaScript не знает, сколько элементов в коллекции в данный момент.
* Размер: Свойства `.size` нет.
* Методы: `set()`, `get()`, `has()`, `delete()`. (Метода `clear()` нет).

#### WeakSet

* Элементы: Только объекты.
* Сборка мусора: Тот же принцип — если на объект внутри `WeakSet` нет внешних ссылок, он удаляется из памяти.
* Перебираемость и размер: Нельзя перебрать, свойства `.size` нет.
* Методы: `add()`, `has()`, `delete()`.



> Интервьюер: _«Приведите пример, когда нужно использовать WeakMap?»_

Ответ: 1. Кэширование данных для объектов. Например, у вас есть объекты пользователей. Вы хотите считать для них сложную статистику и кэшировать её. Если пользователь удаляется из системы (и объект зануляется), кэш в `WeakMap` очистится автоматически.

> Интервьюер: _«Зачем нужен WeakSet?»_

Ответ: Для отслеживания состояния объектов. Например, нужно проверить, посещал ли пользователь определенную страницу (где страница — это объект), или запущен ли уже определенный процесс для этого объекта. Как только сам объект станет ненужным, отметка о нем в `WeakSet` исчезнет.

[^1]: 

    В отличие от обычных объектов, где ключи всегда приводятся к строкам или символам, в `Map` ключом может быть функция, объект или любой примитив.

# В чем разница между setTimeout() и setInterval()

**setTimeout()** — выполняет функцию один раз через указанное время.

**setInterval()** — выполняет функцию многократно, повторяя вызов через каждый указанный интервал, пока не будет остановлена.



Ключевые моменты для интервью:

* Оба возвращают ID (число), по которому можно отменить выполнение через `clearTimeout(id)` / `clearInterval(id)`.
* Оба работают через event loop — не блокируют основной поток, помещаются в очередь макрозадач (macrotask queue) и выполняются после текущего стека вызовов.
* Реальная задержка не гарантирована точно — если основной поток занят, выполнение откладывается.
* Частая ловушка: `setInterval` может накапливать задержки, если сама функция выполняется дольше интервала. Поэтому часто вместо `setInterval` используют рекурсивный `setTimeout` — так можно гарантировать, что следующий вызов запустится только после завершения предыдущего.

# Какие есть основные методы работы с таймерами в JavaScript и как они связаны с коллбеками?

* **setTimeout()** — выполняет функцию один раз через указанное время.
* **setInterval()** — выполняет функцию многократно, повторяя вызов через каждый указанный интервал, пока не будет остановлена.
* **setImmediate** — есть только в Node.js, ставит callback в очередь на выполнение "как можно скорее" после текущей операции ввода-вывода.
* **requestAnimationFrame(callback)** — вызывает callback перед следующей перерисовкой кадра (используется для анимаций, не таймер в чистом виде, но родственная тема).

#### Связь с коллбеками

* setTimeout/setInterval принимают **функцию-коллбек**, которая будет вызвана асинхронно.
* Таймер **не гарантирует точность** времени — delay это минимальная задержка, а не точная. Реальное время выполнения зависит от загруженности event loop.
* Коллбек не выполняется сразу — он попадает в **очередь макрозадач (macrotask queue)** и выполнится только когда:
  1. Истечёт указанное время
  2. Стек вызовов (call stack) станет пустым
  3. Очередь микрозадач (промисы, queueMicrotask) будет пустая

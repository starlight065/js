# В чем разница между стеком вызовов (call stack) и очередью задач (task queue)?

## Call Stack (стек вызовов)

* Структура данных LIFO (Last In, First Out)
* Хранит **синхронный** выполняющийся код: функции, которые сейчас вызваны
* Когда функция вызывается — попадает в стек, когда завершается — удаляется
* Однопоточность JS означает: **один** call stack, выполняется строго последовательно
* Если стек переполнен (например, бесконечная рекурсия) → `Stack Overflow`

```js
function first() {
  second();
}
function second() {
  console.log('hi');
}
first();

// Стек: [first] → [first, second] → [first] → []
```

## Task Queue (очередь задач)

* Структура данных FIFO (First In, First Out)
* Хранит **асинхронные** callback'и, которые ждут своей очереди на выполнение
* Делится на два вида очередей:
  * **Macrotask queue** — setTimeout, setInterval, DOM события
  * **Microtask queue** — Promise.then, queueMicrotask

#### Ключевая разница

|                | Call Stack            | Task Queue                       |
| -------------- | --------------------- | -------------------------------- |
| Что хранит     | Выполняющиеся функции | Callback'и, ожидающие выполнения |
| Порядок        | LIFO                  | FIFO                             |
| Когда работает | Прямо сейчас          | Ждёт, пока стек опустеет         |
| Синхронность   | Синхронный код        | Результат асинхронных операций   |
| Количество     | Один                  | Несколько (micro/macro)          |

#### Как они связаны (роль Event Loop)

Event Loop — это "мост" между ними: он постоянно проверяет, пуст ли Call Stack, и если да — переносит задачу из Task Queue в Call Stack на выполнение.

```js
console.log('A'); // сразу в Call Stack

setTimeout(() => console.log('B'), 0); 
// B уходит в Web API → потом в Macrotask Queue
// Попадёт в Call Stack только когда стек пуст

console.log('C'); // сразу в Call Stack

// Вывод: A, C, B
```

**Короткая формулировка для собеса:**

> Call Stack — это где код _выполняется_ прямо сейчас. Task Queue — это где код _ждёт_, чтобы попасть в Call Stack.

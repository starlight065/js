# Объясните работу цикла событий (event loop) в JavaScript

Проблема: JS однопоточный, но не должен "зависать" при долгих операциях (сетевые запросы, таймеры). Event loop решает, как асинхронный код выполняется без блокировки основного потока.

**Компоненты:**

1. **Call Stack** (стек вызовов) — сюда попадает синхронный код, выполняется по принципу LIFO.
2. **Web APIs / Node APIs** — сюда уходят асинхронные операции (`setTimeout`, `fetch`, DOM события). Браузер/Node сам их обрабатывает вне стека.
3. **Callback Queue (Macrotask queue)** — очередь для callback'ов из `setTimeout`, `setInterval`, DOM-событий.
4. **Microtask Queue** — очередь для промисов (`.then`, `.catch`, `.finally`), `queueMicrotask`.
5. **Event Loop** — постоянно проверяет: пуст ли Call Stack? Если да — берёт задачу из очереди и кладёт в стек.



**Порядок выполнения:**

1. Выполняется весь синхронный код (Call Stack).
2. Когда стек пуст — забираем **все** задачи из Microtask Queue (пока она не опустеет).
3. Забираем **одну** задачу из Macrotask Queue.
4. Снова проверяем микрозадачи.
5. Повторяем.

**Важно:** микрозадачи имеют приоритет над макрозадачами — очередь микрозадач полностью очищается перед каждой макрозадачей.



```javascript
console.log('1');

setTimeout(() => console.log('2'), 0); // Macrotask

Promise.resolve().then(() => console.log('3')); // Microtask

console.log('4');

// Вывод: 1, 4, 3, 2
```

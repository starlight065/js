# В чем разница между использованием обычных промисов (Promises) и синтаксиса async/await?

Разница между Promises и async/await — это разница в синтаксисе, а не в логике. Под капотом `async/await` работает на тех же самых промисах.

#### 1. Читаемость и стиль кода

* Promises: Используют цепочки методов `.then()` и `.catch()`. При большом количестве зависимых запросов код может сильно разрастаться (callback hell на промисах).
* Async/Await: Позволяет писать асинхронный код так, будто он синхронный (линейный). Код читается сверху вниз.

#### 2. Обработка ошибок

* Promises: Ошибки перехватываются через метод `.catch()` в конце цепочки.
* Async/Await: Использует классическую конструкцию `try...catch`, что делает обработку ошибок стандартной для JS.

#### 3. Отладка (Debugging)

* Promises: Сложнее ставить точки останова (breakpoints) внутри цепочек `.then()`, так как это стрелочные функции, и шаг отладчика может перескакивать их.
* Async/Await: Отладка работает идеально. Можно поставить breakpoint на конкретную строку с `await` и пошагово смотреть, как меняются переменные.

#### Сравнительная таблица

| **Критерий**  | **Promises (.then/.catch)**         | **Async / Await**                           |
| ------------- | ----------------------------------- | ------------------------------------------- |
| Синтаксис     | Цепочки методов (Fluent API)        | Ключевые слова `async` и `await`            |
| Вид кода      | Асинхронный (вложенный)             | Выглядит как синхронный (линейный)          |
| Ошибки        | Метод `.catch()`                    | Блок `try { ... } catch (e) { ... }`        |
| Условия/Циклы | Сложно комбинировать внутри `.then` | Легко использовать стандартные `if` и `for` |

#### Главное для ответа:

> `async/await` — это «синтаксический сахар» над промисами. Он не меняет суть асинхронности в JS, но делает код чище, проще для чтения и удобнее для отладки.

# В чем разница между функцией обратного вызова (callback function) и промисом (Promise)?

### Callback

Функция, которую передают в другую функцию, чтобы вызвать её после завершения асинхронной операции.

```js
fetchData(function(err, data) {
  if (err) return console.error(err);
  console.log(data);
});
```

**Проблемы:**

* **Callback hell** — вложенные колбэки превращаются в "пирамиду смерти"
* Сложная обработка ошибок (нужно проверять `err` в каждом колбэке)

### Promise

Объект, представляющий результат асинхронной операции в будущем. Имеет 3 состояния: `pending`, `fulfilled`, `rejected`.

```js
fetchData()
  .then(data => console.log(data))
  .catch(err => console.error(err));
```

**Преимущества над callback:**

* **Цепочки** через `.then()` вместо вложенности — код читается линейно
* **Единая обработка ошибок** через `.catch()` — не нужно проверять ошибку на каждом шаге
* **Гарантии**: промис выполнится ровно один раз, состояние неизменяемо после разрешения
* Можно комбинировать несколько промисов: `Promise.all`, `Promise.race`, `Promise.allSettled`
* Поддержка `async/await` — синтаксический сахар поверх промисов

### Кратко для интервью

|                | Callback                  | Promise                           |
| -------------- | ------------------------- | --------------------------------- |
| Вложенность    | Callback hell             | Плоские цепочки `.then()`         |
| Ошибки         | Проверка в каждом колбэке | Единый `.catch()`                 |
| Состояние      | Нет                       | pending/fulfilled/rejected        |
| Комбинирование | Сложно                    | `Promise.all/race/allSettled`     |
| Читаемость     | Низкая при вложенности    | Высокая, особенно с `async/await` |

Promise — это более современная и удобная абстракция **поверх** того же принципа асинхронности, решающая структурные проблемы callback-подхода.

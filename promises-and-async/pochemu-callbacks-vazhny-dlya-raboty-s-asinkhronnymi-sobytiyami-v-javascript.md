# Почему callbacks важны для работы с асинхронными событиями в JavaScript?

Коротко: JavaScript однопоточный, и асинхронные операции (запросы к серверу, таймеры, чтение файлов) не могут "заморозить" выполнение кода в ожидании результата. Callback — это способ сказать: "когда результат будет готов, выполни эту функцию".

**Суть:**

* Callback — функция, переданная как аргумент в другую функцию, которая вызовется позже, когда асинхронная операция завершится
* Без callback пришлось бы либо блокировать поток (плохо для UX), либо не знать, когда операция закончилась
